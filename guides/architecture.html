Prompts & what each asks the model

spec_gatherer.md "Understand what the user wants and output a structured requirements.json. Ask smart questions, produce valid JSON."
spec_researcher.md "Validate external integrations and dependencies. Verify everything. Trust nothing assumed. Document findings."
architecture.md "You are Winston, Principal Architect. Design ERD, OpenAPI, ADRs, Mermaid diagrams for this requirements set."
spec_writer.md "Read the gathered context and write a complete spec.md. Synthesize context into actionable spec. No user interaction."
spec_critic.md "Critically review spec.md, find issues, fix them. Use extended thinking (max). Find problems BEFORE implementation."
complexity_assessor.md "Analyze the task and determine its true complexity. Accuracy over speed. Wrong complexity → failed implementation."
planner.md "Create a subtask-based implementation plan. Subtasks, not tests. Implementation order matters. Each subtask = one unit of work."
coder.md "You are Alex. FRESH context — no memory of prior sessions. Work on ONE subtask at a time. Complete it. Verify. Move on."
coder_recovery.md "Previous attempts failed. Read attempt_history.json. Try a different approach — don't repeat the same fix."
qa_reviewer.md "You are Morgan. Last line of defense. Verify acceptance criteria via real commands. If you approve, the feature ships."
qa_fixer.md "You are Riley. Fix exactly what QA found. Don't introduce new issues. Get to approval."
validation_fixer.md "This artifact failed schema validation. Repair it. Return only valid JSON."

Subtasks — implementation_plan.json

// stories grouped, each story has subtasks
{
  "stories": [{
    "id": "story-1",
    "title": "User authentication",
    "acceptance_criteria": [
      "User can sign up with email",
      "Session persists across reload"
    ],
    "tech_context": {
      "stack": "Next.js · NextAuth"
    },
    "subtasks": [{
      "id": "sub-1.1",
      "title": "Add /api/auth route",
      "files": ["app/api/auth/*"],
      "verification": {
        "type": "command",
        "run": "npm test auth"
      },
      "status": "completed",
      "attempts": 1
    }]
  }]
}

Status transitions: pending → in_progress → completed · or → stuck after 3 failed attempts. Coder picks the next pending on every iteration via get_next_subtask().

Logs & artifacts

  • spec.mdSpec document written by spec_writer.
  • requirements.jsonStructured user requirements + acceptance.
  • context.jsonRelevant files / patterns / imports discovered.
  • architecture.jsonERD · OpenAPI · ADRs · Mermaid (Lv 3+ only).
  • implementation_plan.jsonStories + subtasks + statuses (single source of truth for coder).
  • build-progress.txtAppend-only line per completed subtask. Streamed to the frontend.
  • memory/attempt_history.jsonPer-subtask attempt log — feeds coder_recovery.md.
  • qa_report.mdQA reviewer verdict (approved / rejected + issues).
  • QA_FIX_REQUEST.mdIssues for the fixer to address (consumed, then deleted).
  • QA_ESCALATION.mdWritten when same issue recurs 3× → human escalation.
  • critique_report.jsonIssues found + fixes applied by spec_critic (ultrathink).
  • complexity_assessment.jsonbmad_level · track · phases_to_run.
  • WebSocket eventsphase:*, task:status, build:progress — emitted by agent_service.py for live UI updates.
